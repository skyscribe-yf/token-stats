# Token Stats Dashboard

A beautiful web dashboard for monitoring AI token usage across multiple providers, built with **Rust (Axum)** backend and **React + Tailwind CSS** frontend.

## Features

- рҹ“Ҡ **Real-time Analytics**: View token usage aggregated by vendor, date, and model
- рҹ“… **Date Range Selection**: Filter data with preset ranges (7D, 30D, 90D) or custom dates
- рҹ“Ҳ **Interactive Charts**: Line charts for daily trends, bar charts for vendor breakdown, pie charts for distribution
- рҹҺҜ **Cache Hit Ratio**: Monitor cache efficiency with weighted and per-request calculations
- рҹ”Қ **Detailed Requests Table**: Paginated view of individual API calls with filterable provider/model
- рҹ’° **Cost Tracking**: Track spending per vendor and model
- рҹҺЁ **Beautiful UI**: Clean, modern design with color-coded vendors and responsive layout

## Data Source

Reads token usage data from `~/.pi/token-logs/usage.jsonl` (generated by the [token-tracker](.pi/agent/extensions/token-tracker.ts) extension).

## Quick Start

### Prerequisites

- Rust (1.70+)
- Node.js (18+)
- nginx (manually installed)

### Automated Setup

```bash
./setup.sh
```

This will:
1. Build the Rust backend
2. Build the React frontend
3. Install nginx configuration
4. Install systemd service

Then start the services:
```bash
sudo systemctl start token-stats
sudo nginx -s reload
```

Access the dashboard at **http://localhost:8080**

### Manual Setup

#### 1. Build Backend

```bash
cd backend
cargo build --release
```

#### 2. Build Frontend

```bash
cd frontend
npm install
npm run build  # Outputs to ../backend/static
```

#### 3. Start Backend

```bash
cd backend
./target/release/token-stats-backend
# Server runs on http://localhost:3000
```

#### 4. Configure Nginx

Copy `nginx/token-stats.conf` to your nginx sites directory:

```bash
sudo cp nginx/token-stats.conf /etc/nginx/sites-available/token-stats
sudo ln -s /etc/nginx/sites-available/token-stats /etc/nginx/sites-enabled/
sudo nginx -t
sudo nginx -s reload
```

## API Endpoints

| Endpoint | Description |
|----------|-------------|
| `GET /api/stats?from=&to=` | Aggregated statistics |
| `GET /api/requests?from=&to=&provider=&model=&page=&limit` | Paginated request details |
| `GET /api/filters` | Available vendors and models |

## Cache Hit Ratio Formula

```
cache_hit_ratio = cacheReadTokens / (inputTokens + cacheReadTokens) Г— 100%
```

- **Weighted ratio**: Aggregated across all requests by total tokens
- **Per-request ratio**: Calculated for each individual API call
